## PURPOSE
I am working in a segmentation project to detect potholes in images of roads. I am using the SAM3 model to generate masks for the potholes, but I need to audit the quality of these masks to ensure that they are accurate and reliable. The orinal annotations provided by the dataset creator are not perfect, so I want to compare them with the masks generated by SAM3 and make a decision on whether to approve or disapprove the SAM3 masks.

I need an app to make the aduitoria of masks generated by sam3 for images in a dataset. The goal is an app to a generic dataset, which contain the orginal image, an bbox or mask provided by dataset creator (which normally is not perfect) and the mask generated by sam3.

We will work with only one class, which is poothole, so do not need to worry about the class of the object, just if the mask is good or not.

## Dataset Structure
The provided dataset will be in a folder structure like this:

```dataset/
├── images/
│   ├── image_name.jpg
├── masks /
│   ├── image_name_mask.png
├── bboxes/
│   ├── image_name_bbox.txt
├── sam_masks/
│   ├── image_name_sam_mask.png
```
OBS: The dataset just need to have one ground truth, which can be the bbox or the mask provided by dataset creator. In case the dataset have both, we will show both in the image comparasion, but with the same color to indicate that they are the same ground truth. If the dataset have only one of them, we will show only the one that is provided.

## App Features
1. **Image Display**: The app will display the original image (whitout any annotation) and besid it, the image with the annotations (bbox or mask provided by dataset creator) and the mask generated by sam3. The color used in each information (bbox/mask by dataset creator and mask by sam3) must be choose by the user, but the coler must be transparent to not hide the original image.

2. **Comands**:
up arrow: its approve the mask generated by sam3, so the name of the image will be saved in the output csv as "approved".

down arrow: its disapprove the mask generated by sam3, so the name of the image will be saved in the output csv as "disapproved".

Right arrow: Nothing

left arrow: its to go back to the previous image, in case the user want to change the decision of approve or disapprove. All images that was returned must have the decision of approve or disapprove removed from the output csv, so the user can make a new decision.

3. **Return Later**: The user can choose to return later to the image, so the name of the image will be saved in the output csv as "return_later". In this case, the image will be shown again in the end of the list of images to be audited.


OBS: if the user exit the app, all the decisions that was made until that moment must be saved in the output csv, so when the user open the app again, it will continue from where it left.

## Output
The output of the app will be a csv file with the name of the image and the decision (approved, disapproved or return_later). The csv file will be saved in root of the dataset folder, with the name "audit_results.csv". The structure of the csv file will be like this:

```csv
image_name,decision
image_name1.jpg,approved

## Format of mask and bbox
The mask is a binary image, where the pixels that belong to the object are white (255) and the pixels that do not belong to the object are black (0). The bbox does not have a format defined, so choose one and make sure to explain in the README file.
